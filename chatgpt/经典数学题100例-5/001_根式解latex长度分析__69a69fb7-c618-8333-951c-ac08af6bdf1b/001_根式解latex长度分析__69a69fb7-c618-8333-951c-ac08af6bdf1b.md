# 根式解LaTeX长度分析

- Conversation ID: `69a69fb7-c618-8333-951c-ac08af6bdf1b`

## Question

我们知道 $\cos \left ( \frac{2\pi }{n} \right ) $ 的根式解是一定存在的（但不一定能尺规作图）。现在我们来探讨其根式解写成LATEX的长度。当n=1000以内的哪个值可以使得其根式解最大？

因为【大】这个东西太空洞了，我们细化一下：
1，只允许使用加减乘除，乘方开方，单位i。不准使用其他符号。
2，我们只采取完全展开式，也就是一行表达式的那种。
3，【大】的量化靠其LATEX的字符串长度衡量。
4，其他细化规则采用相同的标准。比如：$\frac{\sqrt{2} + \sqrt{3}}{2}$ 和 $\frac{1}{2} \left ( \sqrt{2} + \sqrt{3} \right )$ 是同一个意思。我们统一采用其中一种即可。只要规则统一，应该是不影响最终结果的。

## Answer

先说一句“让问题有意义”的必要补丁：在你允许的语法里（只用 $+,-,\times,\div$、乘方开方、$i$），如果不约定“取最短（或某个固定算法输出的）根式表达”，那表达式可以随便加 $+0$、$\times1$、$(a-a)$ 之类把 LaTeX 字符串长度做得任意大，所以“最大”会变成不适定问题。下面我按最常见也最合理的理解：比较每个 $n$ 的“最短根式表达式长度”（或等价地：任何合理固定求根算法产物的长度量级）。

#### 结论（$n\le 1000$ 的“最难”在 $n=719$）

在“用根式把 $\cos(2\pi/n)$ 写出来”这件事上，真正拖累长度的往往不是 $n$ 本身多大，而是你在根式里不得不引入多少层“新的奇素数阶根”以及对应的单位根。原因是：

一旦你需要做某个奇素数 $q$ 次开方（或等价的 Kummer/Lagrange resolvent 步骤），你通常必须同时能写出原始 $q$ 次单位根 $\zeta_q$；

但写出 $\zeta_q$ 又会递归地需要处理 $q-1$ 的素因子（因为 $\mathrm{Gal}(\Bbb Q(\zeta_q)/\Bbb Q)\cong(\Bbb Z/q\Bbb Z)^\times$ 的阶是 $q-1$），于是出现“素数链”：

```math
q \;\Rightarrow\; \text{某个素数 }r\mid (q-1)\;\Rightarrow\;\cdots
```

这条递归链越长，你的根式里不可避免地要出现越多层嵌套结构（不管你怎么“统一格式”，括号、$\sqrt[\;]{\;}$、加减号数量都会被推高）。

把“最长的这种递归链”形式化，正好就是素数的 Pratt 树高度（等价于“证明素数性的 Pratt certificate 深度”）：
对素数 $p$，定义

```math
h(p)=1+\max_{q\mid(p-1)} h(q),\quad h(2)=1.
```

我枚举了所有 $p\le 1000$ 的 $h(p)$，最大者是 $p=719$，且它的链可以显式写出来：

```math
719-1=2\cdot 359,\quad
359-1=2\cdot 179,\quad
179-1=2\cdot 89,\quad
89-1=2^3\cdot 11,\quad
11-1=2\cdot 5,\quad
5-1=2^2.
```

也就是一条很长的“必须递归处理的素数链”：

```math
719 \Rightarrow 359 \Rightarrow 179 \Rightarrow 89 \Rightarrow 11 \Rightarrow 5 \Rightarrow 2,
```

高度 $h(719)=7$，在 $1000$ 以内独一档。于是，在你要求“完全展开、不能用求和符号等压缩记号”的规则下，$\cos(2\pi/719)$ 的根式表达会在“不可避免的嵌套层数/结构复杂度”上压过其它 $n\le 1000$，从而最容易把 LaTeX 字符串长度推到最大。

#### 供你对比的几个“看起来很大但链更短”的竞争者

$n=997$（最大欧拉函数/最大次数的那种直觉）：$\varphi(997)=996$，确实次数最大，但递归链更短（$997-1=2^2\cdot 3\cdot 83$，再到 $83\Rightarrow 41\Rightarrow 5\Rightarrow 2$，高度只有 5），在“根式嵌套不可压缩层数”这件事上不如 719。

$n=771=3\cdot 257$（“全是 2-群所以纯平方根塔”）：$\varphi(771)=512=2^9$，很适合用纯 $\sqrt{\;}$ 链，但它涉及的“奇素数递归链”其实很短（257 是 Fermat 素数，$257-1=2^8$），在这一套“链越长越难压缩”的尺度下也不如 719。

如果你愿意把“最短表达式”进一步完全形式化（比如规定：固定用哪一种 Lagrange resolvent / Gaussian periods 的子群链、固定符号排版规则、固定整数常量的书写方式等），那就可以把它真的变成一个可计算的“字符串长度最大化”问题；但在不引入这些额外约定时，$n=719$ 是最稳的“复杂度冠军”（最长不可避免素数递归链），而 $n=997$ 是“次数冠军”（$\varphi(n)$ 最大）。
