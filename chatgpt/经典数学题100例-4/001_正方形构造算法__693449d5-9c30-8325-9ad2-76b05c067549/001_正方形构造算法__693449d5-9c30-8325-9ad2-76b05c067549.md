## Question

我在平面内取四个点A,B,C,D。分别过A,B,C,D四点作直线，使得这四条直线构成正方形。且A,B,C,D在其边上，不在顶点上，也不在延长线上。能作出这个正方形的充要条件是什么（即A,B,C,D的位置要满足什么关系）？ 请给出一个判定的算法。我举几个例子： ABC构成正三角形，D为其中心点，这四点就无法作出。 ABCD构成一个菱形，∠A=1°，也无法作出。

我这里给一种思路：利用向量 $\vec{AC}$ 旋转 $90^\circ$ 后与 $\vec{BD}$ 的差值来锁定正方形边框的唯一方向。随后在此方向建立坐标系，验证这四个点是否落在生成的正方形边长区间内（即判定是否“出界”）。请证明这种思路的正确性。并用python实现它。多谢！

## Answer

下面我给出必要且充分条件的直观几何表述、对你思路（将 $\vec{AC}$ 旋转90°并与 $\vec{BD}$ 比较以确定正方形方向）的证明要点，以及一个可直接运行的 Python 判定算法（含数值容差与返回构造好的正方形参数）。代码会尝试所有可能的“哪两点为对边”划分（共 3 种），若能找到满足条件的划分，则认为可以作出所需的正方形。

### 结论（必要且充分条件，口语化）

把四点记为 $P_0,P_1,P_2,P_3$。存在一组把这四点两两分为两对「互为对边上的点」的划分（例如 $(P_i,P_j)$ 与 $(P_k,P_\ell)$），使得：

令 $a=\overrightarrow{P_jP_i}$（对边之一的向量），$b=\overrightarrow{P_kP_\ell}$（另一对边的向量，注意顺序可取），则向量

```math
w = R_{90^\circ}(a) - b
```

与某单位向量 $u$ 平行（非零），并且以该 $u$ 作为正方形的一条边方向能推出正方形的边长 $s>0$、中心 $O$ 等参数，且四个点在对应的四条边的“开区间”上（即不在顶点上也不在延长线上）。

换句话说：存在一组把四点配成两对对边，使得上面通过旋转差量得到的方向能一致地重构出一个正方形（并且每个点严格落在某一边的内部）。

这条件既是必要（若确实能画出这样的正方形，按上面配对取相对的两边，推导会给出上式成立）又是充分（若上式成立且满足段内约束，则由上式可以唯一构造出正方形并验证四点落在四条边的内部）。

下面给出推导要点（证明思路），再给出实现。

### 推导要点（说明你给出的思路为何正确）

设正方形的局部坐标系为以中心 $O$ 为原点、单位向量 $u$ 为 x 轴方向、$v=R_{90^\circ}(u)$ 为 y 轴方向，边长为 $s$。我们把四条边按顺序记为：左（x'=-s/2）、上（y'=+s/2）、右（x'=+s/2）、下（y'=-s/2）。若四点按某种分配分别落在左、上、右、下四条边（顺序可循环），那么取对边的两点（左与右）为 $P_0,P_2$，另一对（上与下）为 $P_1,P_3$。在这种标号下有：

$P_2-P_0 = s\,u + \alpha\,v$（这里 $\alpha$ 是两点沿 v 方向的差值）

$P_1-P_3 = s\,v + \beta\,u$（$\beta$ 是沿 u 的差值）

把上式对 $P_2-P_0$ 做 90° 旋转（记作 $R_{90^\circ}$），利用 $R_{90^\circ}(u)=v,\;R_{90^\circ}(v)=-u$ 得

```math
R_{90^\circ}(P_2-P_0)=s\,v - \alpha\,u.
```

因此

```math
R_{90^\circ}(P_2-P_0) - (P_1-P_3) = ( -\alpha-\beta )\,u,
```

即这个向量与 $u$ 平行。也就是说，向量 $R_{90^\circ}(P_2-P_0) - (P_1-P_3)$（也就是你说的“把 $\vec{AC}$ 旋转 90° 后与 $\vec{BD}$ 的差值”）确实给出了正方形一条确定的方向（或其反向）。这就是你思路的正确性证明要点。

从这一方向 $u$ 就可以继续计算出 $s=\langle P_2-P_0,\;u\rangle$（点积，等于向量在 $u$ 方向的分量，即两点跨过的正方形边长），再通过两组对边中点解出中心 $O$（见实现），最后检验四点是否满足“恰好在四条边的内部段上（非端点）”。若满足，则构造成功；若不满足，对其它可能的“对边配对”重复上述过程。

由于四点可能存在三种不同的互为对边的划分（$(01)(23),(02)(13),(03)(12)$），算法需要枚举这三种情况；在每种情况中又自动处理向量方向的符号问题（即 $u$ 或 $-u$ 都被接受，只要最终 $s>0$ 并且四点落在正确的边上）。

### 算法复杂度

枚举 3 种配对，每种配对做常数次向量/点积/检验 => 时间复杂度 $O(1)$。数值上用一个容差（如 $1\mathrm{e}{-8}$）判定近似相等和“是否在开区间内”。

### Python 实现

下面的实现给出一个函数 find_square_from_points(points, tol=1e-8)：

输入 points 为长度为 4 的可迭代对象，每个元素是二元坐标 (x,y)（可以用 NumPy 数组或 Python list/tuple）。

返回 (True, params) 如果存在满足条件的正方形，其中 params 是字典 { 'center':(ox,oy), 'u':(ux,uy), 'side':s, 'assign':assignment }，assignment 指明哪两个点被视作一对对边。

若不可构造则返回 (False, None)。

代码（可直接复制运行）：

```python
import math
def add(a,b): return (a[0]+b[0], a[1]+b[1])
def sub(a,b): return (a[0]-b[0], a[1]-b[1])
def mul(a, k): return (a[0]*k, a[1]*k)
def dot(a,b): return a[0]*b[0] + a[1]*b[1]
def norm(a): return math.hypot(a[0], a[1])
def rot90(a): return (-a[1], a[0])
def normalize(a):
    n = norm(a)
    if n == 0: return (0.0, 0.0), 0.0
    return (a[0]/n, a[1]/n), n
def close(a,b,tol=1e-8):
    return norm(sub(a,b)) <= tol
def find_square_from_points(points, tol=1e-8):
    """
    points: list/tuple of 4 points, each is (x,y)
    return: (True, params) or (False, None)
    params: {'center':(ox,oy), 'u':(ux,uy), 'side':s, 'assign':(i,j,k,l)}
            meaning (i,j) are opposite pair, (k,l) are opposite pair
    """
    pts = list(points)
    if len(pts) != 4:
        raise ValueError("需要4个点")
    # all 3 ways to partition 4 indices into two unordered pairs
    pairings = [
        (0,1,2,3),
        (0,2,1,3),
        (0,3,1,2)
    ]
    for (i,j,k,l) in pairings:
        A = pts[i]; C = pts[j]  # treat as one opposite pair
        B = pts[k]; D = pts[l]  # other opposite pair
        a = sub(C, A)  # vector C - A
        b = sub(B, D)  # vector B - D  (order arbitrary; sign accounted later)
        # compute w = R90(a) - b
        w = sub(rot90(a), b)
        w_norm = norm(w)
        if w_norm <= tol:
            # 无法确定唯一方向（退化），跳过
            continue
        u, _ = normalize(w)   # candidate u (may point either方向)
        v = rot90(u)
        # ensure s = dot(a, u) is positive; if negative flip u
        s = dot(a, u)
        if s < 0:
            u = (-u[0], -u[1])
            v = rot90(u)
            s = dot(a, u)
        # s should be >0 and reasonably large
        if s <= tol:
            continue
        # compute alpha = component of a along v; beta = component of b along u
        alpha = dot(a, v)
        beta  = dot(b, u)
        m = mul(add(A, C), 0.5)  # midpoint of A and C
        n = mul(add(B, D), 0.5)  # midpoint of B and D
        O1 = sub(m, mul(v, alpha*0.5))
        O2 = sub(n, mul(u, beta*0.5))
        if not close(O1, O2, tol=max(tol, s*1e-9)):
            # 中心由两对中点推出不一致 -> 不是同一个正方形
            continue
        O = mul(add(O1, O2), 0.5)
        # 现在验证四个点确实落在四条边的内部（非端点）
        ok = True
        # we expect: A and C lie on x' = -s/2 and +s/2 respectively (within tol)
        # B and D lie on y' = +s/2 and -s/2 respectively (order possibly swapped)
        half = s*0.5
        # compute local coords for each point: x' = dot(p-O, u), y' = dot(p-O, v)
        local = [ (dot(sub(p, O), u), dot(sub(p, O), v)) for p in pts ]
        # helper to check point p is on side x' = target_x and its y' in (-half, half)
        def check_on_x(p_local, target_x):
            x, y = p_local
            if abs(x - target_x) > max(tol, s*1e-9): return False
            if not (-half + max(tol, s*1e-9) < y < half - max(tol, s*1e-9)): return False
            return True
        def check_on_y(p_local, target_y):
            x, y = p_local
            if abs(y - target_y) > max(tol, s*1e-9): return False
            if not (-half + max(tol, s*1e-9) < x < half - max(tol, s*1e-9)): return False
            return True
        # We don't know whether A is left or right, B is top or bottom; check both possibilities.
        # Scenario 1: A on left (x'=-half), C on right (x'=+half); B on top (y'=+half), D bottom (y'=-half)
        sc1 = ( check_on_x(local[i], -half) and check_on_x(local[j], +half)
                and check_on_y(local[k], +half) and check_on_y(local[l], -half) )
        # Scenario 2: flip signs of pairs (A on right etc.)
        sc2 = ( check_on_x(local[i], +half) and check_on_x(local[j], -half)
                and check_on_y(local[k], -half) and check_on_y(local[l], +half) )
        if not (sc1 or sc2):
            # 这种配对下不能把四点放到四条边的内部
            continue
        # 如果都通过，返回参数
        params = {
            'center': O,
            'u': u,
            'v': v,
            'side': s,
            'assign': (i,j,k,l)
        }
        return True, params
    return False, None
# ---------- 测试举例 ----------
if __name__ == "__main__":
    # 能作出的例子：以中心 (0,0)，边长 2，未旋转的正方形，四个点在4条边中间
    pts_ok = [(-1,0.3),(0.4,1),(1,-0.2),(-0.2,-1)]
    ok, p = find_square_from_points(pts_ok)
    print("ok example:", ok, p)
    # 不能作的例子：等边三角形的三个顶点 + 中心（你给的例子）
    # 用一个等边三角形顶点与中心加一个额外点构造测试：
    A=(0, math.sqrt(3)/3)  # 顶点
    B=(-0.5, -math.sqrt(3)/6)
    C=(0.5, -math.sqrt(3)/6)
    D=(0,0) # 中心
    ok2, p2 = find_square_from_points([A,B,C,D])
    print("equilateral + center example:", ok2, p2)
    # 你提到的菱形角度1度的情况也很可能被判为 False
    # (这里仅示例如何调用)
```

### 说明与数值鲁棒性

算法对“退化”情况（例如四点共线、两对向量使得 $w$ 近似零）会返回失败（无法唯一确定方位）。

数值容差 tol 可以按点坐标量级放大或缩小，代码里在比较“是否落在边内部”时还把 s*1e-9 作为相对容差以防止浮点误差把端点判成内部。

算法枚举了 3 种可能的“谁和谁为对边”的划分，理论上这是穷举所有不同的对边组合（没有遗漏）。

若需要返回具体四条边所对应的线段端点或边的方程，函数返回的 center,u,v,side 可以很容易地用矩阵变换重构四条边的边界端点：四条边的中点分别是 $O \pm \frac{s}{2}u$ 和 $O \pm \frac{s}{2}v$，并可沿相应方向截取长为 $s$ 的段。

给出的算法实现了这一定理的构造性检验：枚举配对 \to 由旋转差确定方向 \to 求边长与中心 \to 验证四点是否“在各自边的内部”。
